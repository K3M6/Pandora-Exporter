How to Use

  1. 1
    Go to pandora.com
  2. 2
    Open DevTools: F12 on Windows/Linux or Cmd+Option+I on Mac
  3. 3
    Click the Console tab
  4. 4
    Copy the entire contents of pandora-exporter.js below and paste it into the console
  5. 5
    Press Enter
  6. 6
    Enter your email and password when prompted
  7. 7
    Wait — files download automatically when done. To stop mid-run, type window.__pandoraExportAbort() in the console.

How the Script Works

Click a step to highlight the relevant lines in the script.

1. Domain check
First thing: verify we're actually on pandora.com using a regex that rejects lookalike domains. If not, the script prints an error and exits immediately.
2. Setup: abort, CSRF, sleep
Creates an AbortController so you can cancel mid-export by typing window.__pandoraExportAbort(). Generates a CSRF token using crypto.getRandomValues() for Pandora's double-submit cookie pattern. Defines a cancellable sleep function that cleans up its listener when the timer fires normally.
3. API wrapper with rate limiting
Every API call goes through this function. It sends the CSRF and auth token headers, checks for abort before each request, and if Pandora returns 429 (rate limited), waits the requested time (capped at 60s) and retries up to 5 times.
4. Login
Prompts for your email and password via browser dialogs, authenticates with Pandora over HTTPS, and immediately nulls the password variable. If login fails, logs the raw response for debugging and lists possible causes.
5. Pagination helper
A reusable function that fetches all pages of results from any Pandora endpoint. Keeps requesting with increasing startIndex until a page comes back with fewer items than the page size. Used by stations, thumbs, and collection fetchers.
6. Fetch stations
Retrieves all your stations using the pagination helper. Handles accounts with 250+ stations across multiple pages.
7. Fetch thumbs (concurrent)
Processes 3 stations at a time using a worker pool. For each station, fetches thumbs up and thumbs down in parallel. Tags each thumb with its station name. Shows progress as [12/47] in the console.
8. Fetch collection
Fetches your saved songs, albums, artists, and playlists one type at a time with pagination. Each type uses a different Pandora content code (TR, AL, AR, PL).
9. Sanitize for sharing
Deep-clones the full export and strips internal Pandora identifiers (listenerId, pandoraId, stationToken, CDN URLs, etc.) so the sanitized file is safe to share publicly.
10. Download files
Creates and downloads three files: the full raw JSON, a sanitized JSON (IDs stripped), and a CSV of liked tracks with formula injection protection and a UTF-8 BOM for Excel compatibility. Prints a summary with elapsed time and counts.
11. Cleanup and logout
Runs in a finally block so it always executes, even on error or abort. Removes the global abort function, then sends a logout request using raw fetch() (bypassing the abort-aware API wrapper so it still works after cancellation).

Security

Runs on pandora.com only
The script checks it is running on pandora.com before doing anything. All network requests go to https://www.pandora.com/api and nowhere else. Search the script for fetch( to verify.
Credentials are not stored
Your password is used once to log in over HTTPS, then immediately cleared from memory. The session is logged out automatically when the export finishes.
Verify before running
The script is ~400 lines and fully readable. Never run a minified version. Check that the BASE constant points to https://www.pandora.com/api.
Sanitized export for sharing
The raw JSON may contain internal account IDs. A second file, *_sanitized.json, strips all identifying fields and is safe to share publicly.
